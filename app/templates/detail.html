Sandbox access

To configure cluster access run following snippet, it creates a Kubernetes client context named {{ args.context_name_prefix}}{{ sandbox.namespace }}: 

cat << EOF > kubeconfig-new
apiVersion: v1
kind: Config
clusters:
  - name: {{ args.cluster_name }}
    cluster:
      server: {{ args.cluster_api_url }}
contexts:
  - name: {{ args.context_name_prefix}}{{ sandbox.namespace }}
    context:
      user: dex.{{ args.cluster_name }}
      cluster: {{ args.cluster_name }}
      namespace: {{ sandbox.namespace }}
users:
  - name: dex.{{ args.cluster_name }}
    user:
      exec:
        apiVersion: client.authentication.k8s.io/v1beta1
        command: kubectl
        args:
          - oidc-login
          - get-token
          - --listen-address=127.0.0.1:27890
          - --oidc-issuer-url=https://dex.{{ args.cluster_name }}
          - --oidc-client-id=kubelogin
          - --oidc-use-pkce
          - --oidc-extra-scope=profile,email,groups
EOF
KUBECONFIG="$HOME/.kube/config:kubeconfig-new" kubectl config view --raw > kubeconfig-merged
mv kubeconfig-merged $HOME/.kube/config
rm -fv kubeconfig-new

Developing with Skaffold

To Skaffold develop using one of the template repositories for Flask, Express.js: 

skaffold dev --kube-context {{ args.context_name_prefix}}{{ sandbox.namespace }} --default-repo harbor.{{ args.context_name_prefix}}{{ sandbox.namespace }} --namespace {{ sandbox.namespace }}

Building locally with Docker

In some cases you might want to have Skaffold run 

docker build
locally. In that case you need to copy the Docker registry credentials from the cluster. 
cat ~/.docker/config.json | jq ".auths.\"harbor.codemowers.eu\".auth = \"$(kubectl get secret --context {{ args.context_name_prefix}}{{ sandbox.namespace }} kaniko-secret -o jsonpath='{.data.config\.json}' | base64 -d | jq -s '.[].auths."harbor.codemowers.eu".auth' -r)\"" > .dockerconfig.json && mv .dockerconfig.json ~/.docker/config.json

To reconfigure Skaffold either comment or remove `build.cluster` section in the `skaffold.yaml` file
